# Product

## Register

product

## Users

自托管爱好者与 RSS 重度用户。自己部署 Feedlyra 于个人服务器或家庭 NAS 上，日常用它替代 NetNewsWire / Feedbin / Inoreader 等托管阅读器。使用上下文多为长时间、专注的阅读场景：在桌面端逐篇浏览订阅源、打开长文深入阅读、用自带 AI 做总结/翻译/追问。中英双语使用者，部分用户依赖中文衬线字体获得良好的正文排版。首要任务是「快速扫读 + 沉浸阅读」，次要任务是「与文章对话」。

## Product Purpose

一个自托管、隐私优先的 RSS 阅读器，把「订阅 → 扫读 → 沉浸阅读」与「AI 总结/翻译/对话」收进同一个界面。成功意味着用户能在一个地方完成信息获取与消化，而无需在阅读器、翻译工具、聊天框之间反复横跳；AI 是阅读的延伸而非干扰， BYOK 让用户对密钥与数据有完全掌控。

## Brand Personality

克制 · 精致 · 善待文字

气质对标 Reeder：界面退到文字之后，让排版与正文成为主角。安静、精确、有阅读品味；不张扬、不堆砌装饰、不靠动效吆喝。AI 出现时也像一位安静的助手，而不是一个抢眼的玩具。

## Anti-references

- **SaaS 仪表盘式花哨渐变**：渐变背景、渐变文字、彩色光晕、玻璃拟态卡片。阅读器不需要营销页面的视觉刺激。
- **企业级后台的灰蓝沉闷**：千篇一律的 slate + 中性灰 + 无个性的数据表格观感。Feedlyra 不是内部管理后台。
- **图标+标题+正文重复堆叠的卡片网格**：信息密度撑不起、视觉噪音拉满的模板化列表。
- **每个 section 顶上的小号全大写 eyebrow（ABOUT / PROCESS / FEATURES）**：AI 脚手架的语法残留。
- **喧宾夺主的 AI 入口**：闪烁光标、彩色 typing 光晕、霸占视觉焦点的聊天框。

## Design Principles

1. **文字优先，界面退后。** 排版、行距、字宽是产品本身，不是装饰。任何 chrome（侧栏、工具栏、聊天面板）默认以低对比中性色存在，仅在交互时浮起。
2. **阅读节奏高于信息密度。** 间距服务于扫读 → 阅读 → 对话的连续动作；宁可留白，也不把屏幕塞满。
3. **克制即设计。** 一个主色、一组中性色、三种可切换色调。色彩用于标识状态与可交互，不用于装饰。渐变与玻璃仅在有明确动机时出现。
4. **AI 是安静的合作者。** 摘要、翻译、对话功能在视觉上与正文同源、同色阶，不打断阅读流；动效仅用于反馈状态（生成中、完成、出错）。
5. **为长时间使用而设计。** 这是天天打开、一用数小时的产品：护眼、低噪、暗色模式与减少动画偏好是一等公民，不是补丁。

## Accessibility & Inclusion

目标 WCAG 2.1 AA：正文对比度 ≥ 4.5:1，大号文字 ≥ 3:1。已内置 dark / light / system 三主题与 indigo / amber / forest 三色调切换，为色觉差异提供主色选项。尊重 `prefers-reduced-motion`，键盘可达性覆盖主要导航与阅读操作（j/k 导航、s 收藏、m 切换已读等快捷键）。i18n 支持英文与简体中文。
