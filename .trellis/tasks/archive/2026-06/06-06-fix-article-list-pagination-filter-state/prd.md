# 修复文章列表分页与过滤状态问题

## Goal

修复文章无限滚动、新文章提示、筛选缓存和“全部已读”入口之间的状态不一致，确保加载旧文章不会触发新文章横幅，文章状态变化后分页不漏项，未读/收藏筛选及计数保持正确，同时避免自动滚动标记已读造成列表跳动。

## What I already know

- 用户明确要求解决代码审查发现的第 1、2、3、5 项，不包含第 4 项键盘导航。
- 文章列表使用 TanStack Query `useInfiniteQuery`，服务端使用基于 `page/limit` 的 offset 分页。
- 当前“新文章”检测把所有未确认 ID 当成新文章，因此 `fetchNextPage` 加载的历史文章也会被误报并隐藏。
- 当前 read/star mutation 会修改所有文章缓存中的字段，但不会维护 `read_status=unread` 或 `starred=true` 查询的成员关系和 `total`。
- 未读列表保留已读文章时，服务端 offset 集合已收缩，后续页面会产生跳项。
- “全部已读”按钮当前只检查已经加载并显示的文章，不代表服务端筛选范围内是否仍有未读文章。
- 项目规范要求 React Query 管理服务端分页状态，列表扁平化时按稳定 ID 去重。
- 项目当前没有前端测试框架，质量门为 ESLint、TypeScript/Vite build；后端使用 pytest。

## Assumptions (temporary)

- “新文章”只指第一页顶部相对于已确认第一页新增的文章，不包括加载下一页得到的历史文章。
- 在“收藏”筛选中取消收藏后，当前已展示的文章不立即移出；收藏状态和收藏总数仍立即同步。
- 在“未读”筛选中，打开文章、手动标记已读和自动滚动标记已读后，当前已展示的文章都不立即移出，避免列表跳动。
- 未读计数和后续分页仍必须反映服务端真实状态；展示稳定性不能导致分页边界漏项。
- 不改变后端 API 的鉴权和文章排序语义。

## Open Questions

- 无。

## Requirements (evolving)

### 1. 新文章检测与无限滚动解耦

- `fetchNextPage` 成功后，新增的历史页 ID 必须自动视为已确认。
- 新文章检测只比较后台刷新带来的顶部新增内容。
- 横幅计数只包含真正的新文章。
- 点击横幅后显示新文章并滚动到列表顶部。
- 切换 feed/filter、手动刷新和全部标记已读时不得产生假横幅。

### 2. 分页在过滤集合变化后不漏项

- 标记已读或取消收藏导致服务端过滤集合收缩时，后续加载不能因 offset 位移漏掉边界文章。
- 已加载页面之间仍按文章 ID 去重。
- 方案需覆盖单篇 read/star mutation 和批量滚动标记已读。

### 3. 过滤缓存和计数一致

- 普通“全部文章”缓存只更新 `is_read` / `is_starred` 字段。
- 未读筛选缓存必须更新 `is_read` 和真实 `total`，但当前已展示的已读文章保留到筛选切换、刷新或查询重建。
- 收藏筛选缓存必须更新 `is_starred` 和真实 `total`，但当前已展示的已取消收藏文章保留到筛选切换、刷新或查询重建。
- 收藏侧栏计数必须立即反映收藏/取消收藏结果。
- mutation 失败时不能把错误的成功状态永久留在缓存。

### 5. “全部已读”入口依据服务端范围

- 只要当前 feed 范围仍存在未读文章，就显示“全部已读”按钮，即使当前已加载页全部已读。
- 在全局范围使用 feeds 的未读总数；在单 feed 范围使用该 feed 的 `unread_count`，或使用等价的服务端总数来源。
- 在“收藏”筛选下，按钮语义仍是当前 feed 范围全部已读，不应只依赖当前收藏页。

## Acceptance Criteria (evolving)

- [x] 初次加载 50 篇后滚动加载第 2 页，不显示“50 篇新文章”横幅，旧文章正常追加。
- [x] 后台刷新在第一页顶部新增 N 篇时，显示 N 篇新文章横幅；点击后文章出现且列表滚到顶部。
- [x] 在 100 篇未读文章的场景中，将第一页文章标记已读后继续加载，原分页边界文章不会丢失。
- [x] 取消收藏后，当前收藏列表不移除已展示文章；收藏状态和侧栏收藏计数立即更新，后续查询按真实成员关系显示。
- [x] 将文章标记为已读时，当前未读列表不移除已展示文章；未读总数和后续分页仍正确。
- [x] 打开文章、手动标记已读或自动滚动标记已读时，当前未读列表均不跳动，继续加载也不漏文章。
- [x] 当前已加载文章全部已读但服务端仍有未读文章时，“全部已读”按钮仍显示。
- [x] `npm run lint` 无新增错误，`npm run build` 通过。
- [x] `uv run pytest -q` 通过；如后端分页契约变化，补充相应测试。

## Definition of Done

- 完成上述验收标准对应的实现。
- 对新文章识别、过滤缓存成员变化、分页边界至少增加可自动验证的测试；若不引入前端测试框架，则将纯逻辑提取为可测试函数或采用项目认可的最小测试方案。
- 前端 lint、typecheck/build 和后端测试通过。
- 检查并更新 `.trellis/spec/` 中相关 React Query/文章列表约定。
- 评估回滚：若分页 API 契约变化，旧前端请求仍应得到有效响应，或在同一部署版本内明确同步升级。

## Out of Scope

- 审查项 4：键盘导航只覆盖前 50 篇。
- 文章列表视觉样式调整。
- Feed 抓取、文章内容提取、AI 摘要/翻译逻辑。
- 与本任务无关的全局 React Query 重构。

## Technical Notes

- 主要前端文件：`frontend/src/components/ArticleList.tsx`、`frontend/src/api/hooks.ts`、`frontend/src/components/Sidebar.tsx`。
- 主要后端文件：`backend/app/routers/articles.py`。
- 当前排序：`published_at DESC NULLS LAST, created_at DESC`；若采用游标分页，需要加入稳定且唯一的 tie-breaker（例如 `Article.id`）。
- offset 分页与可变过滤集合天然存在边界位移；推荐采用稳定游标或等价的不可变边界方案，而不是依赖客户端修正 offset。
- 新文章基线应围绕第一页顶部快照管理，不能遍历所有已加载页后把未知 ID 一律判为新文章。
